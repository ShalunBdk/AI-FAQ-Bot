3.4 Тестовые периоды

Назначение

Тестовые периоды позволяют:

  • Оценить качество работы бота за определенный период
  • Сравнить эффективность до и после изменений
  • Получить статистику по группам тестирования
  • Архивировать логи для анализа
  • Экспортировать отчеты

Создание тестового периода

  1. Перейдите в раздел Тестовые периоды
  2. Нажмите "Создать период"
  3. Введите название (например, "Тестовая группа #1")
  4. Добавьте описание (опционально)
  5. Нажмите "Создать"

Важно: Одновременно может быть активен только один тестовый период. Все новые логи будут привязаны к активному периоду.

Просмотр статистики

Для активного или завершенного периода доступна статистика:

  • Всего запросов: общее количество вопросов
  • Успешных ответов: процент найденных ответов
  • Распределение по уровням: exact/keyword/semantic/none
  • Средняя уверенность: средний confidence
  • Неотвеченные запросы: список FAQ, которых не хватает

Жизненный цикл периода

Статус Описание Действия
active Период активен, логи записываются Завершить, Просмотр статистики
ended Период завершен, сбор данных остановлен Архивировать, Экспорт, Статистика
archived Логи архивированы Экспорт, Просмотр статистики

Типичный workflow

1. Подготовка к тестированию

  1. Очистите неархивированные логи (если нужно начать с чистого листа)
  2. Создайте новый тестовый период
  3. Сообщите тестировщикам о начале теста

2. Проведение тестирования

  1. Пользователи задают вопросы боту
  2. Все запросы автоматически связываются с активным периодом
  3. Мониторьте статистику в реальном времени

3. Анализ результатов

  1. Завершите тестовый период
  2. Просмотрите статистику
  3. Изучите неотвеченные запросы
  4. Экспортируйте отчет

4. Улучшение базы знаний

  1. Добавьте недостающие FAQ на основе неотвеченных запросов
  2. Улучшите ключевые слова
  3. Нажмите Retrain
  4. Создайте новый тестовый период для проверки улучшений

Экспорт отчетов

Доступные форматы:

  • Excel (.xlsx) — для анализа в Excel/Google Sheets
  • CSV (.csv) — универсальный текстовый формат
  • JSON (.json) — для программной обработки

Отчет содержит:

  • Все запросы за период
  • Найденные ответы и уровень поиска
  • Confidence (уверенность)
  • Дату и время каждого запроса
  • Платформу (Telegram/Bitrix24)

Очистка логов

Перед запуском в production рекомендуется очистить тестовые логи:

  1. Убедитесь, что все нужные периоды архивированы
  2. Нажмите "Очистить неархивированные логи"
  3. Подтвердите действие

⚠️ Внимание: Очистка удаляет логи безвозвратно. Экспортируйте данные перед очисткой!
